기획

어떤 문제를 왜 해결하려 했는가

소프트웨어 운영 중 발생하는 런타임 에러(Runtime Error)는 서비스 중단의 주된 원인입니다. 특히 무인 서버나 실시간 제어 환경에서는 개발자가 즉각 대응하기 어려워 작은 버그가 치명적인 사고로 이어집니다.

AFIX는 에러 발생 시 시스템이 스스로 Gemini AI를 호출하여 원인을 분석하고, 인간의 개입 없이 즉시 코드를 수정하여 서비스를 지속하는 자가 치유(Self-Healing) 인프라를 구축하고자 기획되었습니다.

구성물

주요 기능 목록

  • 서버 자가 치유 FastAPI 예외 핸들러와 연동된 실시간 코드 수정 엔진
  • Omni-Watchdog 프로젝트 내 모든 파이썬 파일의 에러를 실시간 감시 및 수리
  • 인코딩 최적화 윈도우 환경 특유의 한글 에러 로그 깨짐 방지 시스템
  • 코어 보호 시스템 핵심 파일(watcher, agent 등)에 대한 수정 접근 제한

결과

완성된 프로그램 설명

에러가 포함된 코드를 저장하는 즉시 Watchdog이 이를 감지합니다. AI가 에러 원인을 파악해 수정된 코드를 반환하며, 1초 내에 소스 코드가 자동으로 덮어씌워져 실행이 재개됩니다. 이를 통해 개발자는 버그 수정에 드는 반복적인 시간을 획기적으로 단축할 수 있습니다.

(여기에 프로그램 실행 스크린샷 이미지를 삽입하세요)
특장점
  • 극강의 경량화: 추가적인 데이터베이스나 무거운 UI 없이 핵심 엔진만으로 동작합니다.
  • 범용적 확장성: 특정 파일에 국한되지 않고 프로젝트 폴더 내 모든 `.py` 파일을 관리합니다.
  • 실시간 반응성: 사람이 인지하기도 전에 시스템이 먼저 에러를 고치고 보고합니다.
한계점
  • 논리 오류 파악: 문법적 에러는 완벽히 고치나, 비즈니스 로직상의 깊은 의도 파악에는 한계가 있습니다.
  • 네트워크 의존성: AI 모델 호출을 위해 반드시 외부 API 연결이 필요합니다.
